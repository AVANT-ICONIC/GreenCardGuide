## Allowed without permission
You may:
- refer to the project by name for factual attribution
- say that your project is "based on" or "forked from" this repository
- preserve upstream notices where required by the relevant licenses

## Not allowed without permission
You may not:
- present your fork as the official project
- use the name or logo in app store listings, domains, or marketing in a way that causes likely confusion
- imply sponsorship, certification, partnership, or endorsement
- use confusingly similar branding for a public-facing derivative that serves end users in the same space

## Why this exists
Because this project addresses high-stakes public guidance, users need a clear way to distinguish the maintained official version from stale, modified, or unofficial forks.

Questions about brand use should be directed to the maintainers.
